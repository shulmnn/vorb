# Security Policy

## Reporting a vulnerability

Please do not publish a security vulnerability in a GitHub issue. Email [support@amnios-group.com](mailto:support@amnios-group.com) with the subject “Vorb security report” and include reproduction steps, affected versions, and the impact you observed.

Do not include real provider API keys, private recordings, or sensitive transcripts. A minimal synthetic example is preferred.

We will acknowledge the report, investigate it, and coordinate disclosure when a fix is ready. Security fixes target the latest released version of Vorb.
